# Contributing to EduLens

Hey! Thanks for considering contributing to EduLens. Here's how you can help:

## Found a Bug?

1. Check if it's already reported in [Issues](https://github.com/Harshk031/edulens/issues)
2. If not, open a new issue with:
   - What you were trying to do
   - What happened instead
   - Steps to reproduce it
   - Your OS and Node version

## Want to Add a Feature?

1. Open an issue first to discuss it
2. Fork the repo
3. Create a branch (`git checkout -b cool-new-feature`)
4. Make your changes
5. Test it thoroughly
6. Commit with a clear message
7. Push and open a PR

## Code Style

- Use meaningful variable names
- Add comments for complex logic
- Follow the existing code style
- Run `npm run lint` before committing

## Testing

- Test your changes with different videos
- Try both Groq and LM Studio if possible
- Make sure the UI doesn't break

## Questions?

Feel free to open an issue or reach out. I'm usually pretty responsive!
